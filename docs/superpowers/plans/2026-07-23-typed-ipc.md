# Typed IPC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A narrow, zod-validated IPC surface — channel contracts in `packages/contracts`, pure main handlers over the storage services and vault, a typed `window.recompose` preload bridge — consumed by the renderer through TanStack Query + router loaders, with the providers page as the first real consumer.

**Architecture:** Contracts define every channel as a `{ request, response }` schema pair plus a `{ ok, value } | { ok, error }` result envelope; a pure dispatcher parses inbound and outbound at the boundary. Main handlers are pure functions over injected stores/codec (the `ipcMain.handle` registration and the preload bridge are thin wiring). The renderer gets a `QueryClient` in the router context; loaders warm caches, screens read via `useSuspenseQuery`, writes are mutations + invalidation. No secret-read channel exists anywhere.

**Tech Stack:** zod (contracts), Electron contextBridge/ipcMain.invoke, @tanstack/react-query 5.101.4 + @tanstack/react-query-devtools 5.101.4.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-23-typed-ipc-design.md`.
- Exact pins: `@tanstack/react-query` 5.101.4 and `@tanstack/react-query-devtools` 5.101.4 as devDependencies of apps/desktop (`pnpm add -D -E` — the storage job proved runtime `dependencies` get externalized out of bundles; everything renderer-side stays devDeps like react itself).
- The 7 channels and their schemas are EXACTLY the spec's table; **no secret-read channel** — response schemas are strict and structurally secret-free.
- Every handler result crosses the bridge as the result envelope `{ ok: true, value } | { ok: false, error: { code, message } }` with codes exactly `vault-unavailable | vault-newer-schema | validation-failed | storage-failed`.
- Boundary validation both directions: the dispatcher parses inbound with the request schema (bad input → `validation-failed`, never a throw across the bridge) and outbound with the response schema.
- Vault debts close here: codec constructed only after `app.whenReady` (inside the registration wiring); `isEncryptionAvailable() === false` or `basic_text` fallback → `vault-unavailable`; `loadVaultFile`'s newer-schema throw → `vault-newer-schema`.
- FSD (Steiger-enforced): renderer query/mutation definitions live in `pages/providers/api/` (page-local); the result-unwrap helper opens `shared/api/`; route adapters stay thin and import pages ONLY via slice public APIs.
- Skills: implementers touching renderer data code read the `tanstack-query` skill first (installed under the main checkout's `.agents/skills/tanstack-query/SKILL.md`; the controller supplies the resolved path in dispatches).
- TDD: failing test first; state-based; doubles ONLY at process boundaries (fs = real temp dirs; safeStorage codec = injected fake; `window.recompose` in browser specs = fake bridge — the preload boundary).
- TypeScript max strictness; no `any`, no silencing `as`. **Never write code comments.**
- The repository owner's private alias must not appear in any artifact.
- Commit messages: Conventional Commits, terse, imperative; trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- Pre-commit hooks run all gates; oxfmt may reformat and auto-stage. If the FIRST commit in a fresh worktree fails with `oxfmt: No such file or directory`, run `pnpm install` once and retry.
- All commands run from the worktree root. Shell may be fish: `echo "exit: $status"`, never after a pipe.

## Execution deviations

- `@tanstack/react-query`/devtools shipped 5.101.3, not the pinned 5.101.4 — `minimumReleaseAge` quarantined the fresher patch; no exclude added; recorded in ADR-0018.
- The dispatcher shipped with sender validation: `dispatchIpc` takes the reduced sender + allowed origins and rejects untrusted senders BEFORE schema parsing (mid-flight spec addition; Task 2 plan text predates it); recorded in ADR-0018.
- The preload bridge shipped as a literal 7-entry enumeration, not the loop derivation shown in Task 3 — cast-free loop derivation is impossible under the lint/TS regime; recorded in ADR-0018.

```ts
import { z } from 'zod';

import { accountKindSchema, accountsDocumentSchema } from './accounts';
import { gatewayConfigSchema } from './gateway-config';
import { nonBlankString } from './non-blank';
import { settingsSchema } from './settings';

export const ipcErrorSchema = z.strictObject({
  code: z.enum(['vault-unavailable', 'vault-newer-schema', 'validation-failed', 'storage-failed']),
  message: z.string().min(1),
});

export type IpcError = z.infer<typeof ipcErrorSchema>;

export function ipcResult<Value extends z.ZodType>(value: Value) {
  return z.union([
    z.strictObject({ ok: z.literal(true), value }),
    z.strictObject({ ok: z.literal(false), error: ipcErrorSchema }),
  ]);
}

export const connectAccountRequestSchema = z.strictObject({
  provider: nonBlankString,
  kind: accountKindSchema,
  label: z.string().trim().min(1),
  secret: nonBlankString,
});

export const ipcChannels = {
  'gateways:list': { request: z.void(), response: ipcResult(z.array(gatewayConfigSchema)) },
  'gateways:save': {
    request: gatewayConfigSchema,
    response: ipcResult(z.array(gatewayConfigSchema)),
  },
  'settings:get': { request: z.void(), response: ipcResult(settingsSchema) },
  'settings:save': { request: settingsSchema, response: ipcResult(settingsSchema) },
  'accounts:list': { request: z.void(), response: ipcResult(accountsDocumentSchema) },
  'accounts:connect': {
    request: connectAccountRequestSchema,
    response: ipcResult(accountsDocumentSchema),
  },
  'accounts:remove': {
    request: z.strictObject({ id: nonBlankString }),
    response: ipcResult(accountsDocumentSchema),
  },
} as const;

export type IpcChannel = keyof typeof ipcChannels;
export type IpcRequest<Channel extends IpcChannel> = z.infer<
  (typeof ipcChannels)[Channel]['request']
>;
export type IpcResponse<Channel extends IpcChannel> = z.infer<
  (typeof ipcChannels)[Channel]['response']
>;

export type RecomposeIpc = {
  [Channel in IpcChannel]: (request: IpcRequest<Channel>) => Promise<IpcResponse<Channel>>;
};
```

Create `apps/desktop/src/main/ipc/dispatch.ts`:

```ts
import {
  ipcChannels,
  type IpcChannel,
  type IpcRequest,
  type IpcResponse,
} from '@recompose/contracts';

export type IpcHandlers = {
  [Channel in IpcChannel]: (request: IpcRequest<Channel>) => Promise<IpcResponse<Channel>>;
};

export async function dispatchIpc(
  handlers: IpcHandlers,
  channel: IpcChannel,
  rawPayload: unknown,
): Promise<unknown> {
  const contract = ipcChannels[channel];
  const parsed = contract.request.safeParse(rawPayload);
  if (!parsed.success) {
    return contract.response.parse({
      ok: false,
      error: { code: 'validation-failed', message: parsed.error.message },
    });
  }
  const handler = handlers[channel] as (request: unknown) => Promise<unknown>;
  const result = await handler(parsed.data);
  return contract.response.parse(result);
}
```

If the `as (request: unknown) => Promise<unknown>` narrowing offends the no-silencing-`as` rule under oxlint, replace with a generic inner function — the requirement is: no cast may weaken the PUBLIC signatures; internal dispatch plumbing may use a single well-contained widening if the compiler cannot express the correlated types (document which form shipped in your report).

```ts
import { randomUUID } from 'node:crypto';
import { join } from 'node:path';

import type { AccountsDocument, IpcError } from '@recompose/contracts';

import { loadAccountsFile, saveAccountsFile } from '../storage/accounts-store';
import { listGatewayConfigs, saveGatewayConfig } from '../storage/gateway-store';
import type { SecretCodec } from '../storage/safe-storage-codec';
import { loadSettingsFile, saveSettingsFile } from '../storage/settings-store';
import { deleteSecret, loadVaultFile, saveVaultFile, setSecret } from '../storage/vault';
import type { IpcHandlers } from './dispatch';

export type StorageIpcContext = {
  userDataPath: string;
  getCodec: () => SecretCodec;
  isEncryptionAvailable: () => boolean;
  onCorrupt: (quarantinedPath: string) => void;
};

function failure(code: IpcError['code'], message: string) {
  return { ok: false as const, error: { code, message } };
}

function storageFailure(error: unknown) {
  return failure(
    'storage-failed',
    error instanceof Error ? error.message : 'storage operation failed',
  );
}

export function createStorageIpcHandlers(ctx: StorageIpcContext): IpcHandlers {
  const gatewaysDir = join(ctx.userDataPath, 'gateways');
  const settingsFile = join(ctx.userDataPath, 'settings.json');
  const accountsFile = join(ctx.userDataPath, 'accounts.json');
  const vaultFile = join(ctx.userDataPath, 'vault.bin');

  async function readAccounts(): Promise<AccountsDocument> {
    return loadAccountsFile(accountsFile, ctx.onCorrupt);
  }

  return {
    'gateways:list': async () => {
      try {
        return { ok: true, value: await listGatewayConfigs(gatewaysDir, ctx.onCorrupt) };
      } catch (error) {
        return storageFailure(error);
      }
    },
    'gateways:save': async (config) => {
      try {
        await saveGatewayConfig(gatewaysDir, config);
        return { ok: true, value: await listGatewayConfigs(gatewaysDir, ctx.onCorrupt) };
      } catch (error) {
        return storageFailure(error);
      }
    },
    'settings:get': async () => {
      try {
        return { ok: true, value: await loadSettingsFile(settingsFile, ctx.onCorrupt) };
      } catch (error) {
        return storageFailure(error);
      }
    },
    'settings:save': async (settings) => {
      try {
        await saveSettingsFile(settingsFile, settings);
        return { ok: true, value: await loadSettingsFile(settingsFile, ctx.onCorrupt) };
      } catch (error) {
        return storageFailure(error);
      }
    },
    'accounts:list': async () => {
      try {
        return { ok: true, value: await readAccounts() };
      } catch (error) {
        return storageFailure(error);
      }
    },
    'accounts:connect': async (request) => {
      if (!ctx.isEncryptionAvailable()) {
        return failure('vault-unavailable', 'OS secret encryption is unavailable');
      }
      let vault;
      try {
        vault = await loadVaultFile(vaultFile, ctx.onCorrupt);
      } catch (error) {
        return failure(
          'vault-newer-schema',
          error instanceof Error ? error.message : 'vault schema is newer than supported',
        );
      }
      try {
        const credentialRef = `cred-${randomUUID()}`;
        const account = {
          id: `acc-${randomUUID()}`,
          provider: request.provider,
          kind: request.kind,
          label: request.label,
          credentialRef,
        };
        await saveVaultFile(
          vaultFile,
          setSecret(vault, ctx.getCodec(), credentialRef, request.secret),
        );
        const accounts = await readAccounts();
        const updated = { ...accounts, accounts: [...accounts.accounts, account] };
        await saveAccountsFile(accountsFile, updated);
        return { ok: true, value: updated };
      } catch (error) {
        return storageFailure(error);
      }
    },
    'accounts:remove': async (request) => {
      try {

        };
        await saveAccountsFile(accountsFile, updated);
        return { ok: true, value: updated };
      } catch (error) {
        return storageFailure(error);

  };

import type { IpcError } from '@recompose/contracts';

export class IpcResultError extends Error {
  readonly code: IpcError['code'];

  constructor(error: IpcError) {
    super(error.message);
    this.code = error.code;
  }
}

export function unwrapIpcResult<Value>(
  result: { ok: true; value: Value } | { ok: false; error: IpcError },
): Value {
  if (!result.ok) {
    throw new IpcResultError(result.error);
  }
  return result.value;
}

# ADR-0018: Typed IPC — Contracts-Defined Channels with a Result Envelope

**Status**: Accepted
**Date**: 2026-07-23

## Context

The storage foundation (ADR-0016) had no consumer and the router skeleton (ADR-0017) had no data. Connecting them required a main↔renderer surface that preserves the repo's structural guarantees: schemas as the single source of truth, secrets never readable from attacker-reachable territory, expected failures as typed states.

## Decision

- **Channel contracts live in `packages/contracts`** (`src/ipc.ts`): seven channels, each a `{ request, response }` zod pair; main handlers, the preload bridge, and renderer types all derive from the one map. A hand-rolled bridge was chosen over electron-trpc: at seven channels, tRPC's router/transformer indirection buys nothing the contracts map doesn't already give, and costs a dependency plus serialization overhead.
- **Result envelope everywhere**: every response is `{ ok: true, value } | { ok: false, error: { code, message } }` with the closed code set `vault-unavailable | vault-newer-schema | validation-failed | storage-failed`. Expected failures cross the bridge as data the renderer can branch on; only contract violations throw.
- **Boundary validation both directions**: a pure dispatcher parses inbound payloads (bad input → `validation-failed`, never an exception across the bridge) and outbound results (drift is structurally impossible). Handlers are pure functions over injected stores and codec; `ipcMain.handle` registration and the preload bridge are thin, coverage-excluded wiring.
- **No secret-read channel exists.** `accounts:connect` carries plaintext inbound exactly once; main generates `id` + `credentialRef`, encrypts into the vault, and returns the secret-free registry. `accounts:remove` deletes the vault entry and account row together. Response schemas are strict — a secret structurally cannot ride back.
- **Vault debts closed**: the safeStorage codec is constructed lazily per operation after `app.whenReady`; `isEncryptionAvailable() === false` maps to `vault-unavailable`; a newer vault schema maps to `vault-newer-schema`.
- **Renderer data layer is TanStack Query over router loaders**: `QueryClient` in the router context, loaders warm caches via `ensureQueryData`, screens read `useSuspenseQuery`, writes are mutations + invalidation. Query definitions are page-local (`pages/<slice>/api/`); the envelope unwrap helper opened `shared/api`. Query devtools mount dev-only under the same gating as the router devtools.

## Alternatives

- **electron-trpc**: full router/procedure ergonomics, but a transformer dependency and indirection tax disproportionate to a seven-channel surface; revisit only if the channel count grows past what a flat map can carry.
- **Exceptions across the bridge**: Electron serializes thrown errors lossily and the renderer cannot branch on them reliably; the envelope keeps failure a typed value.
- **A secret-read channel for the renderer**: would simplify future edit flows but hands plaintext to the most exposed process; rejected permanently — the engine receives secrets by in-memory handoff instead.

## Consequences

**Good**: adding a channel is one contracts entry plus one handler — types, validation, and the bridge follow; the renderer cannot receive malformed data or leak secrets structurally; vault failures render as actionable UI states; the providers screen is a real end-to-end consumer proving the whole path.

**Bad**: seven explicit preload entries must track the channel map by hand (typecheck catches drift); the envelope adds one unwrap layer to every renderer call; push-style updates remain unsolved until the engine lands.
```
