# ADR-0018: Typed IPC — Contracts-Defined Channels with a Result Envelope

**Status**: Accepted
**Date**: 2026-07-23

## Context

The storage foundation (ADR-0016) had no consumer and the router skeleton (ADR-0017) had no data. Connecting them required a main-renderer surface that preserves the repo's structural guarantees: schemas as the single source of truth, secrets never readable from attacker-reachable territory, expected failures as typed states, and every inbound invocation verified as coming from the app's own window before it reaches a handler.

## Decision

- **Channel contracts live in `packages/contracts`** (`src/ipc.ts`): seven channels, each a `{ request, response }` zod pair — `gateways:list`, `gateways:save`, `settings:get`, `settings:save`, `accounts:list`, `accounts:connect`, `accounts:remove`. Main handlers, the preload bridge, and renderer types all derive from the one map. A hand-rolled bridge was chosen over electron-trpc: at seven channels, tRPC's router/transformer indirection buys nothing the contracts map doesn't already give, and costs a dependency plus serialization overhead.
- **Result envelope everywhere**: every response is `{ ok: true, value } | { ok: false, error: { code, message } }` with the closed code set `vault-unavailable | vault-newer-schema | validation-failed | storage-failed`. Expected failures cross the bridge as data the renderer can branch on; only contract violations throw.
- **Boundary validation both directions**: the dispatcher parses the inbound payload with the channel's request schema — a malformed payload returns `validation-failed`, never an exception across the bridge — and parses the handler's result with the response schema before it leaves main. The outbound `.parse()` throws a runtime `ZodError` the moment a handler returns the wrong shape, and the dispatch specs assert that throw for every channel — so the mistake surfaces as a failing test, not a silent shape mismatch on the wire. Symmetry was chosen over validating only the trust-crossing direction because the outbound parse is also where the no-secret-read invariant becomes structural, not just a matter of handler discipline.
- **Sender validation runs before schema parsing, inside a testable dispatch layer.** The dispatcher takes a sender derived from `event.senderFrame`: the frame is null-checked (frames can be disposed mid-flight), must be the invoking window's own main frame, and its URL's origin must match one of the app's own origins — the packaged build's `file://` origin or the dev-server origin, never a wildcard. A rejected sender throws a bare, uninformative `Error('untrusted ipc sender')` rather than returning a typed result code: this is deliberate, following Electron's own IPC guidance not to hand a potentially hostile sender any structured signal about why it was rejected. It is the one path on this surface where an expected failure is not a typed result. Residual risk, recorded honestly: `file://` is a broad trusted origin — anything the packaged app ever loads from that origin passes — and Electron recommends a custom app protocol for tighter scoping. That hardening is deferred to the security-baseline job (queue item 7), not solved here.
- **No secret-read channel exists.** `accounts:connect` carries plaintext inbound exactly once; main generates `id` and `credentialRef`, encrypts into the vault, and returns the secret-free registry. `accounts:remove` deletes the vault entry and the account row together. Response schemas are strict objects with no secret field — a secret cannot ride back structurally. The invariant extends to failure paths: no error branch of `accounts:connect` serializes the secret into `error.message`, and nothing on that channel's path logs the request payload.
- **Preload is a literal enumeration, not a loop over the channel map.** The bridge lists all seven channels by hand, then freezes the result object. A loop-derived bridge was tried and is provably impossible to type cast-free under this repo's lint/TS regime: TypeScript's structural narrowing loses the per-key correspondence between channel and handler signature (TS2740, missing properties), and closing that gap needs an `as` cast, which the oxlint `no-unsafe-assignment` rule then flags. Totality is instead compile-time via the `RecomposeIpc` type — the frozen bridge object fails to typecheck if a channel is missing, extra, or mistyped. Preload's imports from `@recompose/contracts` are type-only, so the electron-vite preload bundle carries no runtime zod.
- **Renderer data layer is TanStack Query over router loaders**: `QueryClient` lives in the router context; route loaders warm caches via `ensureQueryData`; screens read with `useSuspenseQuery`; writes are mutations followed by `invalidateQueries`. Query and mutation definitions are page-local (`pages/providers/api/`), per Pages-First — promotion to a shared layer waits for a second consumer. The providers page is the first real consumer, replacing its placeholder list with the accounts registry.
- **`@tanstack/react-query` and its devtools are pinned to `5.101.3`, one patch behind the `5.101.4` available at implementation time.** ADR-0015's `minimumReleaseAge` policy (3 days) quarantined the newer patch. Rather than add a `minimumReleaseAgeExclude` entry — the escape hatch ADR-0015 documents for a deliberately-fresh add — the older, already-matured version was kept: the less invasive path when nothing this job needs depends on the newer patch.
- **Query devtools mount dev-only**, gated identically to the router devtools already in the root route (dev-mode check, excluded under vitest, dynamically imported). Verified against the built production bundle: zero occurrences of the devtools module names.

## Alternatives

- **electron-trpc**: full router/procedure ergonomics, but a transformer dependency and indirection tax disproportionate to a seven-channel surface; revisit only if the channel count grows past what a flat map can carry.
- **Exceptions across the bridge**: Electron serializes thrown errors lossily and the renderer cannot branch on them reliably; the envelope keeps failure a typed value instead.
- **A secret-read channel for the renderer**: would simplify future edit flows but hands plaintext to the most exposed process; rejected permanently — the engine receives secrets by in-memory handoff instead.
- **A typed `forbidden-sender` result instead of a thrown error**: the design left this open. Rejected once implementation confirmed Electron's guidance against giving a hostile sender a structured signal to probe against — an uninformative throw is the safer default here, even at the cost of breaking the "expected failures are typed results" pattern used everywhere else on this surface.
- **Loop-derived preload bridge**: would remove the seven-line hand enumeration, but only by introducing an `as` cast the lint regime rejects; literal enumeration plus compile-time totality was kept instead.

## Consequences

**Good**: adding a channel is one contracts entry, one handler, and one preload line — the compiler flags a missing or mistyped handler via `IpcHandlers` and a missing or mistyped preload entry via `RecomposeIpc`; the main-process registration list is a plain array the compiler can't verify for completeness, so a test asserts it covers every contract channel exactly, matching what the compiler can't; the renderer cannot receive malformed data or leak secrets structurally, in both success and failure paths; a foreign or disposed frame never reaches a handler, and learns nothing about why; vault failures render as actionable UI states; the providers screen is a real end-to-end consumer proving the whole path.

**Bad**: seven explicit preload entries must be kept in sync with the channel map by hand — a missing or extra one is a compile error, not silent drift, but still manual; `file://` stays a broad trusted origin until the security-baseline job narrows it; the envelope adds one unwrap layer to every renderer call; the sender-rejection throw is the one place on this surface where failure isn't a typed code, which future readers need to recognize as deliberate rather than an oversight; `@tanstack/react-query` sits one patch behind latest until it clears the release-age window naturally; push-style updates remain unsolved until the engine lands.
